# Contribution Guidelines

Please note that this project is released with a [Contributor Code of Conduct](https://www.contributor-covenant.org/version/2/1/code_of_conduct/). By participating in this project you agree to abide by its terms.

## Adding an entry

Ensure your pull request adheres to the following guidelines:

- **WebMCP only.** Only submit resources directly about the WebMCP browser standard (`navigator.modelContext` API or `toolname`/`tooldescription` HTML attributes). Do not submit general server-side MCP resources.
- **Quality bar.** The project should have a README, contain actual code or meaningful documentation, and provide value to the WebMCP community.
- **One link per pull request.** If you want to add multiple entries, send separate PRs so they can be reviewed independently.
- **Format.** Use the format `- [Name](url) - Description ending with a period.` Descriptions should be concise (one sentence) and not include promotional language.
- **No star counts or badges inline.** These go stale quickly.
- **Alphabetical order** within subsections is preferred but not required; relevance ordering is acceptable.
- **Check your spelling and grammar.**
- **New categories** are welcome if the entry doesn't fit any existing section, but justify the addition in the PR description.

## Updating an entry

- If a project has moved, changed names, or become archived, open a PR to update or remove it.
- If a description is inaccurate or outdated, corrections are welcome.

## Reporting issues

If you find a broken link, outdated description, or misplaced entry, open an issue describing the problem.

Thank you for contributing!
